第1章 概论

网络的类型及其特征

根据网络的拓扑结构分类：

  • 星型结构，存在一个中心结点
  • 树形结构
  • 总线型结构，存在一条主干线
  • 环型结构，存在一个环形的总线
  • 网络型结构，不规则的连接

1.4 计算机通信协议与网络体系结构

网络协议，是一套语义和语法规则，规定了网络中使用的格式、定时方式、顺序和检错。

网络协议的组成：

  • 语法，“怎么讲”，数据与控制信息的结构或格式
  • 语义，“讲什么”，解释协议元素的含义
  • 同步，规定事件的执行顺序

协议层次模型。

协议中包含实体和接口：

OSI-RM 体系结构

是一种计算机网络层次结构模型，分层结构，遵循协议分层的原则。

包括了体系结构、服务定义和协议规范三级抽象：

  • 体系结构方面，定义了一个七层模型
  • 服务定义方面，描述了各层所提供的服务、层之间的抽象接口、交互用原语
  • 协议规范方面，定义应当发送何种控制信息、何种过程来解释该控制信息

七层模型，从低到高：

  • 应用层，包含使用的各种协议
  • 表示层，信息的表示方式处理，包括格式变换、加密解密、压缩恢复等
  • 会话层，维护通信中两个结点之间的会话连接的建立、维护和断开，负责数据交换
  • 传输层，向用户提供端到端的数据传输服务，屏蔽底层
  • 网络层，为数据包选择适当的路由，实现拥塞控制、网络互连等功能
  • 数据链路层，在物理层提供服务的基础上，在通信的实体间建立数据链路链接
    • 传输以帧为单位的数据包
  • 物理层，数据传输

发送方从上到下传递到物理媒体，接收方从下到上处理消息。

从上到下层传递的过程中，每层都要加上适当的控制信息。

OSI-RM 模型只是一个概念性框架，并非具体实现。

TCP/IP 体系结构

包含四个层次（从低到高）：应用层、传输层、互连网络层、网络接入层。

通信子网中的处理设备只需要最低二层的功能。

2.6 差错控制技术

四类差错控制方式：

  • 自动请求重发，接收到错误信息时要求发送端重发
  • 前向纠错，通过计算确定差错的位置，自动加以纠正
  • 混合方式，少量错误自动纠错，超过纠正能力则重发
  • 信息反馈，接收端把收到的数据序列全部发回发送端，发送端把错误的数据序列的原始数据再次发送

海明码，又称为汉明码。具有纠错能力。

  • 海明码需要的位数：若信息位 kkk，额外 rrr，则应满足 2r≥k+r+12^r≥k+r+12r≥k+r+1
  • 记作 (n,k)(n,k)(n,k)
    • 意义：用额外位表示所有码元的状态，以确保一位纠错能力
  • 校验码的位置：在最终编码的位置为 2i−1,i=1,2,…2^{i-1},i=1,2,…2i−1,i=1,2,…
  • 海明码的获得：
    • 首先确定各个位与各个校验码的关系（MMM 表示最终编码的对应位）：
      • M1M_1M1​ 表示成 0001，则只与第一个校验码 e1e_1e1​ 有关；
      • M2M_2M2​ 表示成 0010，则只与第二个校验码 e2e_2e2​ 有关；
      • M3M_3M3​ 表示成 0011，则与 e1e_1e1​、e2e_2e2​ 有关；
      • ……
    • 然后对于各个校验码，将与它有关的 MMM 编码位用异或符号 ⨁\bigoplus⨁ 连接计算：
      • e1=M1⨁M3⨁M5⨁M7⨁M9e_1=M_1\bigoplus M_3\bigoplus M_5\bigoplus M_7\bigoplus M_9e1​=M1​⨁M3​⨁M5​⨁M7​⨁M9​
      • ……
    • 如果海明码没有错误信息，各个校验码 eee 都为 0；若有错，例如 S3=1S_3=1S3​=1 其余都是 0，则 c3c_3c3​ 有误
    • 例，101101 的海明码为 0010011101

注意，计网里的海明码，监督位统一排在最后面。

循环冗余校验码 CRC。

  • 接收端和发送端约定一个“除数”，信息码和校验码作为“被除数”，校验时判断余数是否为 0
  • 二进制的除法：各个位取异或
  • 构造方法：
    • 确定编码长度。根据生成多项式，取最高次幂次数 rrr 为校验码位数，获得其编码
    • 信息码左移 r−1r-1r−1 位。信息码在最左侧
    • 对移位后的信息码，除以生成多项式的编码，获得余数
    • 将余数与左移过的信息码相加，得到结果
  • 检错：
    • 接收方用生成多项式的编码去除得到的编码，若余数为 0，则没有错误

第3章 数据链路层

3.1 数据链路层

数据链路层，将原始的物理线路改进成逻辑上无差错的数据链路。

信道，信息传输的通路。或模拟或数字信道。
数据电路，由传输信道、DCE 组成。
数据链路，数据电路建立的基础上，实现正确通信的组合体。

站，链路上所连接的结点：
主站，发送命令或信息的站。
从站，接收数据或命令的站。
复合站，同时具有主站和从站功能，能够发出命令和响应信息的站。

帧定界，网络层的分组必须封装成帧：一段数据前后分别添加首部和尾部。

3.2 流量控制和差错控制

流量控制

控制发送方发送数据的速率，使接收方来得及接收。

几种方式：

停止-等待流量控制，每发送一帧，必须等到接收方发出应答信号，才发送下一帧

  • 过程：
    • 从网络层取一个数据帧，将发送序号写入帧内，然后通过物理层发送，并设置超时时间
    • 接收帧序号需要与接收到的相匹配
  • 性能相关：
    • 传播时延 tpt_ptp​，与距离信号速度有关
    • 发送时延 tst_sts​，与数据发送速率和数据帧长度有关
    • 处理时延
    • 信道利用率 = 发送时延 / 从发送到应答所需时间

滑动窗口流量控制，发送端和接收端各有一个能存放多帧的窗口，完成一帧就向后滑动，窗口之外的帧就不发送/不响应

  • 每一帧会额外发送序号，让接收方分辨重复帧，让发送方分辨应答帧对应的帧
  • 连续 ARQ 协议，返回 N 帧协议：
    • 发送窗口尺寸 WTW_TWT​ 大于 1，接收窗口尺寸等于 1
    • 若用 n 个比特进行编号，发送窗口尺寸必须小于等于 2n−12^n-12n−1
  • 选择 ARQ 协议，选择重传协议：
    • 接收窗口尺寸 WRW_RWR​ 不等于 1
    • 若用 n 个比特进行编号，发送窗口尺寸取最大值时 WR=WT=2n−1W_R=W_T=2^{n-1}WR​=WT​=2n−1

3.3 点对点信道的数据链路层协议

HDLC

变相比特的传输控制规程。

两种基本配置：

  • 非平衡配置，由一个主站控制整个链路的工作
  • 平衡配置，链路两端的两个站都是复合站，每个复合站都可以平等地发起数据传输

HDLC 帧格式：

  • 长帧，F - A - C - I - FCS - F
  • 短帧，F - A - C - FCS - F

其中：

  • F，标志字段
    • 指明每一帧的开始和结束
    • 共 8 位：连续 6 个 1，加上头尾两个 0（01111110）
    • 暂无信息发送时，可连续发送 F，作为帧间填充，同时用于保持收发双方的同步
    • 若非 F 段出现 01111110，发送时在 5 个 1 后插入一个 0（011111010）
  • A，地址字段
    • 一般 8 位
  • C，控制字段
    • 根据控制字段的不同，可把帧分为信息帧 I、监督帧 S、无编号帧 U
  • 信息字段
    • 只有信息帧 I 有信息字段
  • 帧校验序列

HDLC 帧的控制字段 C 标识了三种帧类型：

  • 信息帧 I
  • 监控帧 S
  • 无编号帧 U

点对点协议

PPP，Point-to-Point Protocol。

3.4 多路访问信道的数据链路层

信道共享技术，动态信道分配方法。

多路访问技术可分为：

  • 受控访问，各个用户不能任意接入信道，必须服从一定的控制
    • 根据控制结构的位置，又分为集中式控制和分散式控制：
      • 集中式控制，多点线路轮询，即控制主机按一定顺序逐个询问各用户有无信息发送
      • 分散式控制，在环路中有一个特殊的帧，叫做令牌或权标。令牌沿环路驻站传递，只有获得令牌的站才有权发送信息；信息发送完毕后，将令牌传递给下一个站。环形网也叫做令牌传递环，是一种常用的局域网
  • 随机访问，所有用户根据自己的需求随机地发送消息。系统会有通信冲突，被称为竞争系统

竞争系统的介质访问控制技术：

  • ALOHA，一种完全随机发送数据的无线局域网
    • 纯 ALOHA，网络中多个站点随机发送数据，产生“冲突”
    • 时隙 ALOHA，网络中的站点只在时隙开始时发送数据，减少冲突的可能性
  • CSMA，ALOHA 的改进
    • 每个站点在发送分组前，监听公共信道是否正忙（先听后说）
      • 非坚持型，当监听到信道空闲时，立即发送，否则就延迟一个随机事件重新监听
      • 1-坚持型，当监听到信道忙时，一直坚持监听，直到监听到空闲，立即发送
      • P-坚持型，当监听到信道忙时，一直坚持监听，直到监听到空闲，以 P 的概率发送，以 1 - P 的概率延迟一个时间单位
    • 仍然不能完全避免冲突
    • CSMA/CD，在发送过程中持续监测有无冲突发生，若有就立即停止发送

环形网戒指访问方法，属于分散的轮询控制方式：

4.2 以太网技术

以太网采样总线式拓扑结构。

以太网的 MAC 层，可唯一表示主机地址的标识符。
MAC 帧格式：

  • 目的地址（DA），6
  • 源地址（SA），6
  • 数据类型字段（T）或数据长度字段（L），2
  • 用户数据字段（DATA），0-1500
  • 填充字段（PAD），0-46
    • 保证整个帧的长度不小于 64 字节
  • 帧校验序列（FCS），4

CSMA/CD 的工作过程：

  • 发送方：
    • LLC 帧传给下层的 MAC 协议实体，MAC 协议实体将 LLC 帧封装在用户数据字段，形成 MAC 帧
    • MAC 协议实体监听传输媒体，检查是否有信号正在传输
    • 如果有信号在传输，则持续监听；否则，发送数据，并继续监听媒体
    • 如果发送过程没有检测到冲突，则本次发送完成；否则，立即终止本次发送过程，向媒体发送一个冲突加强的信号，以使其他结点都能感知到发送冲突，MAC 协议实体计算发送失败的次数
    • 若失败次数小于等于某个阈值，根据失败次数执行二进制指数退避算法，计算退避时间，准备重新发送；否则，通知上层 LLC 实体可能出现网络故障
  • 接收方：
    • 监听媒体，接受长度大于最小有效帧长的帧，分析帧中的目的地址

争用期，一台计算机发送数据，最多经过端到端往返时延 2τ2τ2τ 时间，就可以知道是否发生了冲突。也称为冲突窗口。
寻找最小帧长，就用 争用期 × 数据速率。
最大帧长，受最大传送单元（MTU）的限制。除了最后一个分片 , 每个分片的长度是 8 字节的整数倍。

4.3 局域网的扩展

在物理层扩展局域网：使用集线器，直接连接局域网。

在数据链路层扩展局域网：使用网桥。

  • 网桥
    • 工作原理：两个端口，接上两个网段。每接受到一个帧就暂存入缓存，若帧无差错并且 MAC 地址属于另一个网段，则转换并转发帧
  • 透明网桥
    • 支撑树算法，避免转发路径存在回路
  • 源路由网桥
    • 源站在发送帧时将详细的路由信息放在帧的首部中
  • 以太网交换机
    • 直通交换方式：直接转发
    • 存储转发方式：根据帧头部的目的 MAC 地址进行转发
    • 无碎片交换方式：收到 64 字节以后才开始以直通方式转发帧

5.1 网络层概念

5.2 网络互连

IP 编址和 IP 数据报支持 TCP/IP 软件隐藏物理网络细节，使互联网看起来是一个统一实体。

最初的分类编址方案包含 5 种形式的 IP 地址：

  • A类，8位网络号，24位主机号
  • B类，16位网络号，16位主机号
  • C类，24位网络号，8位主机号

为了方便，采用点分十进制记法书写 IP 地址。例如，某主机 32 比特的 IP 地址：
10000001 00000001 01000110 00001111
可写成 129.1.70.15。

IP 地址

分为两个部分：前缀（网络号）、后缀（主机号）。

前缀越长，支持的物理网络越多；后缀越长，支持的主机越多（更大规模的物理网络）。

因特网地址到物理地址的映射

48 位的以太网地址编入 32 位的 IP 地址不太可行。物理地址要额外找方法。

地址统一（网络层以上使用 IP 地址）的代价是，需要建立 IP 地址和物理地址的映射。

ARP 协议，地址解析协议，思路是广播一个特殊报文，某个 IP 地址的某个主机响应后响应它的物理地址报文。实现 IP 对 MAC 地址的映射。

ARP 表，ARP 表在高速缓冲中，存放最近获得的 IP 地址与物理地址的映射。一般设有超时定时器，例如 20 分钟。

IP 数据报

IP 数据报（简称数据报）是因特网的基本传送单元，它提供对物理网络帧的统一。

与典型的物理网络帧类似，有首部和数据区。首部中包含的是 IP 地址，而物理帧的是物理地址。

数据报被封装在物理帧中，作为帧的数据传送。对于以太网，帧类型为 0x0800 表示帧数据区存放的是 IP 数据报。

数据报首部包含 20 字节（32×8 比特）的固定部分，和可选的 IP 选项部分：

  • 版本，4 bit
  • 首部长度，4 bit。IP 首部最大长度为 15×4 字节，数据报首部长度必须是 4 字节的整倍数，最常见为 20 字节，字段值为 5
  • 服务类型（TOS），8 bit。如何处理数据报
  • 总长度，16 bit
  • 标识，16 bit。源主机赋予数据报的唯一标识符，一段时间内是唯一的
  • 标志，3 bit
  • 片偏移量，13 bit

无连接的数据报传送

5.3 差错与控制报文协议（ICMP）

辅助 IP 协议。

  • ICMP 目的不可达报文
    • 代码描述问题：
      • 0，网络不可达
      • 1，主机不可达
      • 2，协议不可达
      • 3，端口不可达
      • 4，需要分片但 DF 被置位
      • 5，源路由失败
  • ICMP 超时报文
    • 格式与目的不可达报文相同，但类型值为 11
    • 代码说明超时的性值：
      • 0，运输过程中 TTL（time to live）超时
      • 1，分片重装超时
  • ICMP 源抑制报文
    • 格式与目的不可达报文相同，但类型值为 4
    • 只有一个代码 0

5.5 因特网的路由选择协议

路由表是怎样初始化和动态更新的。

整个因特网被划分为许多自治系统（AS）。因特网可以看过是随意链接的 AS 的集合。

因特网路由选择协议可划分为两大类，

  • 内部网关协议 IGP
  • 外部网关协议 EGP

内部网关协议 RIP

路由信息协议 RIP，是一种内部网关协议，常用于小型的自治系统。RIP 一条路径最多 15 跳。

RIP 使用跳数测度，这样，最佳路由就是能够以最小跳数到达某目的的网络的路由。

路由表的更新：

  • 首先初始化为与自己直接相连的每个网络生成一个表项
  • 每个路由器，根据相邻路由器定义发来的路由信息，更新自己的路由表，获悉更多目的网络以及到各网络的最佳路由
    • 若目的网络 D 不在路由表中，则将 (目的网络 D, 距离 M+1, 下一跳 Y) 加到路由表中

内部网关协议 OSPF

运用于自治系统内部，使用链路状态算法（或称最短路径优先算法）。

每个路由器使用 Djikstra 最短路径算法，构建一个以自己为根的最短路径树（SPF 树），计算得出路由表。

OSPF 能提供负载均衡功能。若有若干代价相同的路由，则把流量平均地分配给这些路由。这与 RIP 不同。

为了能够运用于大规模网络，OSPF 允许将一个自治系统（AS）中的网络分成若干组，每个组称为一个区域。划分区域的好处是，通过洪泛法交换链路信息的范围被局限，减少了整个网络的通信量。

OSPF 直接使用 IP 数据报传送，而不使用 UDP 或 TCP。所以它位于网络层。

外部网关协议 BGP

BGP 是一个自治系统之间的路由选择协议。所以，一个 AS 若只有一条到因特网或另一个 AS 的连接，就不需要使用 BGP。

BGP 只能是力求寻找一条能够到达目的网络且比较好的路由。确保 AS 无环地交换路由选择信息。

特点：

  • BGP 发言系统的主要功能，是与其他 BGP 系统交换称为路径向量的网络可达性信息
  • 经由 GBP 交换的路由选择信息仅支持基于目的的转发模式
  • BGP 提供一组机制，支持无类域间路由
  • BGP 假定一个 AS 内部的路由选择由 IGP 完成，对自治系统使用什么 IGP 没有特别要求，对自治系统之间的互连拓扑不作限制

5.6 IP 组播

IP 组播是对硬件组播的互联网抽象。仍然表示到达一个主机子集的传输，但概念更广泛，它允许主机子集跨越互联网上任意的物理网络。

比起单播，IP 组播下源主机只需要发送一个数据报，剩下的多转发功能由路由完成。

IP 组播需要把 D 类地址（224.0.0.0 ~ 239.255.255.255）作为目的地址。IP 组播地址只可作为目的地址。

第6章 传输层

传输层，在通信子网提供服务的基础上为上层提供传输服务，高层用户在通信时不必关心通信子网的实现细节和具体服务质量。

6.1 传输服务

传输层的功能

网络层协议能将分组送达目的主机，但无法交付给主机中的应用进程。换句话说，网络层负责提供通信，传输层建立一条端到端的逻辑通信信道。

传输层会对整个报文段进行差错校验和检测，包括首部和数据部分。

当传输层采用面向连接的协议（如 TCP）时，它为应用进程在传输实体间建立一条全双工的可靠逻辑信道。当传输层采用如 UDP 这样的无连接协议时，这种逻辑信道是不可靠的。

传输层的存在使得传输服务比网络服务更加合理有效。对于 TCP，分组的丢失、残缺甚至网络重置都可以被传输层检测到，并采取相应的补救措施。如果应用层协议强调实时应用或者广播、多播等要求，那么选择 UDP 为宜。

传输层采用一个标准的原语集提供传输服务。由于传输服务独立于网络服务，故可以采用一个标准的原语集提供传输服务。传输服务是标准的，它为网络想高层提供了一个统一的服务界面，所以用传输服务原语编写的应用程序就可以广泛适用于各种网络。

传输层编址

传输层的 UDP 和 TCP 都使用端口，与上层的应用进程进行通信。

传输层与应用层的接口上所设置端口是一个 16 位的地址。TCP 和 UDP 各有 65536 个端口可以使用。

端口只有本地意义。

传输层的端口有两种类型：

  • 熟知端口，专门分配给一些最常用的应用层程序，数值为 0~1024，由 TCP/IP 体系确定并公布
  • 一般的端口号，用来随时分配给请求通信的客户进程

无连接服务和面向连接服务

无连接的服务 面向连接的服务
通信之前不需要建立连接 需要建立连接、保持连接，通信完毕后释放连接
数据按顺序发送，但微必按顺序接收 按顺序接收
不可靠服务 可靠服务
协议简单，效率高 协议复杂，效率不高

6.2 无连接的传输层协议 UDP

UDP，用户数据报协议，在 IP 数据报服务之上增加了端口复用与分用和差错控制的功能。其特点如下：

  • 传输数据前不需要与对方建立连接
  • UDP 提供不可靠的服务，数据可能不按发送顺序到达接收方，也可能会重复或者丢失数据
  • UDP 同时支持点到点和多点之间的通信
  • UDP 是面向报文的。发送方的 UDP 对应用程序提交的报文，在封装成 UDP 用户数据报后就向下交付给网络层处理；接收方的 UDP 对网络层交上来的 UDP 用户数据报，去除首部之后就递交给应用程序。UDP 适于传输短的报文数据

UDP 首部由 4 个字段组成，每个字段都是 2 字节：

  • 源端口字段，标识源端口号
  • 目的端口字段，标识目的端口号
  • 长度字段，UDP 数据报的长度，以字节为单位，包括首部的 8 个字节
  • 检验和字段，防止传输中出错

6.3 面向连接的传输层协议 TCP

其特点如下：

  • TCP 是面向连接的，在通信之前双方必须建立 TCP 连接
  • TCP 提供可靠的服务，可以保证传输的数据按顺序到达，且不出差错、不丢失、不重复
  • TCP 只能进行点到点的通信
  • TCP 面向字节流，发送方的 TCP 将应用程序交下来的数据视为无结构的字节流，分割成 TCP 报文段进行传输

TCP 首部格式

应用层的报文传送到传输层，加上 TCP 的首部，就构成 TCP 的数据传送单元，称为报文段。

TCP 报文段首部的前 20 个字节是固定的。详见 IP 数据报。

  • 源端口，2 字节
  • 目的端口，2 字节
  • 序号，4 字节
  • 确认号，4 字节
  • 数据偏移，2 bit，指出 TCP 报文段的首部长度，以 4 字节为单位
  • 标志位，用于区分不同类型的 TCP 报文
    • URG，报文段中包含紧急数据
    • ACK，确认号字段有效
    • PSH，应尽快将此报文交付给接收应用程序
    • RST，表名 TCP 连接出现严重差错，须释放连接，然后再重新建立连接
    • SYN，在连接建立时用来同步序号
    • FIN，用来释放一个连接

若是 IP 数据报，需要从第 21 个字符读到 TCP 首部。

TCP 连接管理

TCP 是面向连接的协议，需要建立和释放传输连接。

TCP 采用三次握手建立连接，

  • 主机 A 向服务器 B 的 TCP 发出连接请求报文段，首部中的同步比特标志位 SYN=1
    • 选择一个初始序号 seq=x
  • 服务器 B 发回确认，标志位 ACK=1
    • 确认号为 ack=x+1，选择一个初始序号 seq=y
  • 主机 A 的 TCP 向服务器 B 确认，ACK=1
    • 确认号为 ack=y+1

通信双方都可以发出释放连接的请求。

TCP 可靠传输

TCP 是可靠的传输层协议，主要通过确认机制和超时重传机制来实现可靠传输。

确认机制，

  • 将整个应用层报文看成是一个个字节组成的数据流，对每一个字节编一个序号。连接建立时确定初始序号。每次传送，将序号放在 TCP 首部的序号字段中
  • TCP 的确认，是对接收到的数据的最高序号（即接收的最后一字节）表示确认。返回的确认序号是最高序号加 1

超时重传机制，

  • TCP 采用一种自适应算法计算重传定时器的定时设置，
    • 记录每一个报文段的往返时延，将各个报文段的时延样本加权平均，得到 RTT。每测量到一个新的往返时延样本，就重新计算依次平均往返时延
    • RTT新值 = α × RTT旧值 + (1-α) × 新的RTT样本

TCP 流量控制

TCP 采用大小可变滑动窗口的方式进行流量控制。窗口大小的单位是字节。

TCP 拥塞控制

避免网络发生拥塞，或者缓解已经发生的拥塞。

拥塞窗口，衡量网络的拥塞程度。发送方的发送窗口的上限值，应取接收方窗口和拥塞窗口这两个值中较小的一个。

几个技术，

  • 慢启动，在 TCP 刚建立连接或者当网络发生拥塞超时的时候，将拥塞窗口 cwnd 设置成一个报文段大小，并且当 cwnd ≤ ssthresh（门限值） 时，指数方式增大 cwnd
  • 拥塞避免，当 cwnd ≥ ssthresh 时，以线性方式增大 cwnd
  • 快速重传，发送方如果连续收到对同一报文段三个重复确认的 ACK，则立即重传该报文段，而不必等待重传定时器超时后重传
  • 快速恢复，当采用快速重传算法时，直接执行拥塞避免算法。提高传输效率

网络拥塞严重时，cwnd=1；若能收到 3 个冗余的 ACK，cwnd 减半。
