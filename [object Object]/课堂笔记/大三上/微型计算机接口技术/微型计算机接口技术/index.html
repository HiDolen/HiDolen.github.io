微型计算机接口技术

第一章 基础

  • 计算机系统组成
    • 冯·诺依曼的计算机体系：
      • 存储器、运算器、控制器、输入设备、输出设备
  • 存储器的一些基本术语

    • 存储元，存储器的最小组成单位，存储 1 位二进制代码

    • 存储单元，CPU 访问存储器的基本单位，由存储元组成

    • 存储体，存储单元的集合，存放二进制信息

    • 单元地址，表示存储单元的编号，CPU 通过编号访问存储单元

    • 存储器，存储体和地址译码电路、读写控制电路等一起构成存储器

      综上，有：

      存储元 → 存储单元 → 存储体 → 存储器

  • 存储容量计算
    • 存储容量=字数×字长。如，字长 8 位，能存储 1024 个字，则容量 8k
  • 存储器分类
    • SRAM，静态 RAM
    • DRAM，动态 RAM
    • ROM，只读存储器
    • PROM，可编程 ROM。只能写一次
    • EPPROM，可擦除 ROM
    • Flash，闪存
  • 存储系统
    • 分级：寄存器 → Cache → 主存储器 → 辅助存储器
  • 存储器的组织
    • 通过地址总线、控制总线、数据总线与 CPU 连接
      • 地址线，决定存储容量
      • 数据线，决定字长为多少位
  • 存储器地址译码方式
    • CPU 地址分为高位地址和地位地址两部分，分别进行片选和片内译码

第五章 总线

  • 总线分类

    • 地址总线，数据总线，控制总线

  • 总线基本概念
    • 性能指标
      • 总线宽度（多少位）；总线频率；传输率（=宽度/8*频率），单位MB/S
    • 总线的信息传送方式
      • 串行传输、并行传输、分时传输
  • 一些举例
    • D/C‾\rm D/\overline{C}D/C，表示：1 时传输的是数据，0 时传输的是代码
    • M/IO‾\rm M/\overline{IO}M/IO，表示：1 时CPU与存储器交换信息，0 时CPU与I/O接口交换信息

第六章 存储系统

  • 存储器的扩展
    • 1B=8bit\rm 1B=8bit1B=8bit，字节与位
    • 一块存储芯片的容量可用 2n×m2^n×m2n×m 位表示。其中，nnn 是地址总线数，mmm 是数据总线数
    • 地址线是用来找地址的，数据线是用来一根一根接到位上的

第七章 输入输出系统

  • 接口电路

    • 功能：
      • 数据缓冲，解决高速主机与低速外设数据速率差异的矛盾
      • 联络功能，向CPU系统提供外设的状态，或通知外设CPU系统的状态，有时能控制外设
      • 寻址功能，接口电路应有I/O端口地址编码器
      • 数据转换功能，计算机系统内部以并行方式处理数据，有的外设只能串行数据通信交互
      • 中断管理功能，接口电路设置中断控制电路，允许或禁止接口电路提出中断请求

  • 输入输出端口

    • 接口电路通常包含一组 寄存器，实现接口的各项功能，称为I/O端口寄存器，简称为端口。能与CPU交换信息

    • 按端口寄存器存放信息的物理意义，区分为 3 类：

      • 数据端口，存放数据信息。数据端口连接数据总线，外设和CPU进行通信
      • 状态端口，存放状态信息，反映外设当前工作状态的信息。CPU可读取这些信息
      • 控制端口，存放CPU通过接口传送给外设的控制信息。是一种输出数据

    • 只有输入指令（IN）和输出指令（OUT）

    • I/O端口的编址方式

      • 端口和存储单元统一编址

        • 把I/O端口作为存储器的一个单元来看待

      • I/O端口独立编址

        • I/O端口和存储器分别使用两个地址空间，单独编址和译码
        • PC系列机，I/O端口采用独立编址方式

  • 输入输出指令

    • I/O端口与CPU（AL或AX）的信息交换

      • 直接寻址的指令

        • 端口地址为一字节（8b）时，采用直接寻址

      • DX间址的指令

        • 端口地址为二字节时，采用间接寻址。端口地址存放在 DX 中

      • 注意事项：

        • 只能用寄存器 AL、AX、EAX 与端口交换数据
        • 用 DX 间接寻址，不加方括号 []

  • 微机系统与 I/O 端口的信息交换

    • 有四种方式：无条件、中断、查询、DMA

      • 无条件传送方式，CPU 直接使用 IN 或 OUT 指令完成与接口之间的数据传送

        • CPU 执行输出指令时，必须保证锁存器时空闲的

      • 查询方式，接收数据前 CPU 查询输入数据是否准备好，输出数据前 CPU 查询输出设备是否空闲

        • 查询方式输入（假设状态口地址200H，D7=1时可输入，数据口地址201H）：

          使用 TEST 和 JZ，不断跳跃。

      • 中断控制方式，外设做好传送准备后向 CPU 发出中断请求，CPU 响应中断请求转入中断服务子程序完成数据的传入。其余时间可执行与传送数据无关的指令

      • DMA 方式，使用专门的 DMA 控制器 代替 CPU 控制总线

        • CPU 不参与操作，传送速度快，但硬件电路比较复杂。DMA方式通常用来传输数据块
        • 实现 DMA 传送需要器件 DMA 控制器，DMAC（Direct Memory Access Chip）
        • DMA 传送与中断方式的比较：CPU 接到 中断请求 后需要等到当前指令执行完毕才响应，而 CPU 接到 DMAC 的 总线请求 后，当前指令的当前总线执行完毕就响应
I/O 设备信息交互方式 优点 缺点
无条件传送方式 可以直接使用输入缓冲器或锁存器与数据线相连，程序设计简单 传送不能太频繁，须保证每次传送都设备处在就绪状态
查询方式 比无条件方式可靠，时间确定 降低 CPU 工作效率，不具有实时性
中断控制方式 提高 CPU 工作效率，具备实时性，可并行工作，不用反复查询外设的工作状态 每次进行数据传输都要保护现场，时间不确定、有抖动
DMA 方式 按数据块传输，不经过 CPU，不需要保护现场 硬件更复杂

第八章 中断系统

  • 中断的概念

    • CPU 由于某种外部或内部事件，被暂停当前执行的程序转去为该事件服务，处理完后返回被中断的程序

  • 中断源

    • 引发中断的事件。可以是外部事件，也可以是 CPU 内部事件

  • 中断的分类

    • 外部中断，也称为硬件中断，由 CPU 以外的设备发出

      • 硬件中断的优先级： DMA 请求 > 非屏蔽中断 > 可屏蔽中断

      • 可分为：可屏蔽中断、非屏蔽中断

      • 可屏蔽中断，通过 8259 中断控制器连接至 CPU 的 INTR。

        • 当 CPU 处于 开中断状态 时，可屏蔽硬件中断请求由外部中断控制器将相应的中断类型码送给 CPU
        • 通过 8259 中断控制器连至 CPU 的 INTR

        • 日时钟中断、键盘中断、用户中断

        • 非屏蔽中断，由 NMI 引脚输入。对于非屏蔽硬件中断请求，CPU 内部会自动产生中断类型码 2

      • 对于 INT n 指令，中断类型码即为指令中给定的 n

    • 内部中断，也称为软件中断

      • INT n
    • 硬件中断和软件中断的区别：
      • 中断的引发方式不同
      • CPU 获取中断类型码码的方式不同
      • CPU 响应的条件不同
      • 中断处理程序的结束方式不同

  • 可屏蔽中断

    • 响应可屏蔽中断的条件：
      • 有可屏蔽中断，没有 DMA 请求，没有非屏蔽中断
      • CPU 一条指令执行完毕
      • CPU 处于开中断状态
    • 中断优先级从高到低：
      • 中断源：日时钟中断、键盘中断、（从8259的IR0~7，有用户中断）、辅串口、主串口、并行口2、软盘、并行口1
      • 中断类型码：08H、09H、70H、71H改向0AH
        • 主8259的IR0对应日时钟，IR1对应键盘；从8259的IR1对应用户中断
    • 日时钟中断

      • 中断类型：08H

      • 中断源：系统 8254 0# 计数器（0号计数器），每 55ms 一次中断请求

    • 键盘中断

      • 中断类型：09H

      • 中断源：主板键盘接口电路
    • 用户中断

      • 中断类型：71H 改向 0AH（用主8259的0AH扩展）

      • 中断源：系统的 ISA 总线 B4 端子
  • 中断指令
    • STI，set i，开中断指令。F 寄存器中 I 标志置 1，CPU 响应可屏蔽中断。只对可屏蔽中断请求有效
    • CLI，clear i，关中断指令。F 寄存器中 I 标志置 0，CPU 不响应可屏蔽中断。只对可屏蔽中断请求有效
    • INT n，软件中断指令。无条件转向 n 型中断服务子程序
      • F 寄存器入栈
      • 断点地址入栈。段基址 CS 和偏移地址 IP
      • CPU 从 4n ~ 4n+3 单元取出 n 型服务程序入口地址
    • IRET，中断返回指令。依次从栈顶弹出 6 个元素，IP、CS、F（偏移、段基址、F）
      • interrupt return
  • 中断类型码，区分不同类型的中断。取值范围 0~255
    • 0 型中断，除法错中段——当除法结果溢出或者除数为 0 时，发生的中断
    • 2 型中断，非屏蔽硬件中断
    • 5 型中断，屏幕打印
    • 08H~0FH 型中断，可屏蔽硬件中断
    • 10H~1FH 型中断，BIOS 中断
  • 中断向量和中断向量表
    • 中断向量，中断服务子程序的 入口地址。每个终端号对应的中段向量占 4 字节

    • 中断向量表，在实模式下，CPU 把 256 种中断向量组成一张中断向量表，设置在系统的 RAM 最低端的 1K 单元（0 ~ 3FFH），每个中断向量占 4 字节

      • n 型中断向量存放在内存单元地址 4n ~ 4n+3 的 4 个单元字节之中。低 2 位放偏移地址，高 2 位放段基址
      • 计算时，因为是 16 进制，所以转成 2 进制进行移位运算会更准确
    • 中断向量表的初始化
      • 由 BIOS 设计的中断服务子程序（如 INT 16H，INT 10H），其中断向量在加电时，由 BIOS 负责写入中断向量表
      • 由 DOS 设计的中断服务子程序（如 INT 21H），其中断向量是在启动 DOS 时，由 DOS 负责写入中断向量表
      • 用户程序开发的中断服务程序

第九章 串行通信

  • 串行通信的定义：数据按时间先后依次一位一位传送。比并行通信速率低，但占用通信线较少

  • 串行通信分为串行异步通信和串行同步通信。重点是串行异步通信

    • 串行异步通信

      • 数据传输方式：
        • 单工通信，只允许数据按照一个固定的方向传送
        • 半双工通信，收发双方均具备接收和发送数据的能力，但只有一条信道，不能同时双向传送
        • 全双工通信，收发双方可同时进行数据传送
      • 特点：通信速率一致，数据帧格式一致
      • 数据帧格式
        • 起始位，1 位，是一位逻辑 0
        • 数据位，5 ~ 8 位，从最低位开始发送
        • 奇偶校验位，1 位或 0 位，使得数据位和奇偶校验位的逻辑 1 的总个数为奇数个或偶数个
        • 停止位可以是 1 位、1.5 位或 2 位的逻辑 1 信号
        • 到下一个字符传输之前可有任意空闲间隔，用逻辑 1 信号表示。两个字符的间隔是任意的，因此被称为串行异步通信

  • RS-232C 接口标准

    • 逻辑 1 信号，用电平 -3V ~ -15V 表示
    • 逻辑 0 信号，用电平 +3V ~ +15V 表示
    • 就是说，1 和 0 是反过来的

异步通信接口芯片 8250

  • 8250 内部寄存器
    • 发送保持寄存器、接受缓冲寄存器，用于实现数据传输
    • 控制字寄存器，用于工作方式、通信参数的设置。有通信线控制寄存器、除数寄存器等
    • 4 级中断及优先级（从高到低）
      • 接收线路状态中断
      • 接收缓冲器满
      • 发送保持器空
      • MODEM 状态中断
    • 状态寄存器
  • 初始化编程

    • 首先确定是主串口还是辅串口。主串口的端口开头为 3F，辅串口的端口开头为 2F

    • （固定）通信线控寄存器中，使寻址位为 1（第 8 位为 1，其他都为 0，即 80H），能够访问除数寄存器

    • 确定通信速率，存入除数寄存器（高8位低8位）

    • 通信线控寄存器中，规定一帧数据格式，同时使除数寄存器访问位=0

      • 一般前两位都是 0

    • 设置中断方式，写入中断允许控制字

    • 设置 MODEM 控制寄存器。一般是全 0（查询方式）；若有中断，D3=1；若是内环自发自收，则 D4=1

    • 开头 I8250 PROC，结尾 RET、I8250 ENDP

    • 端口号速记：最后一位，B 9 8 B 9 C

  • 应用编程
    • 查询状态：使用通信线状态寄存器。只有 D1=1 时，CPU 才能读数；只有 D5=1 或 D6=1 时，CPU 才能写数据

第十章 并行 I/O 接口

8255A 定时器/计数器

可编程的并行接口芯片。

  • 端口
    • 题目不会直接给出端口地址。但根据题目给出的首地址，可以依次为 4 个端口：
      • A口、B口、C口、控制口
    • 端口编址分别为：A1A_1A1​、A0A_0A0​、RD‾\rm\overline{RD}RD、WR‾\rm\overline{WR}WR、CS‾\rm\overline{CS}CS。其中，A1A_1A1​ A0A_0A0​ 用于分别选 4 个寄存器

  • 内部结构

    • 8255A内部有 3 个 8 位的输入输出端口：A口、B口、C口。从内部控制的角度，可分为 A组、B组
      • A组，A口和C口的高四位
      • B组，B口和C口的低四位
    • 数据总线缓冲器，与系统数据总线相连，实现 CPU 和 8255A 端口之间的信息交换
    • 读写控制逻辑模块
    • A组控制模块、B组控制模块
    • I/O 端口

  • 工作方式

    • 方式 0：基本型输入输出方式
      • 基本型输入输出，端口和外设之间不需要联络信号
      • A口、B口、C口 适用
      • INTEA，A 口中断允许寄存器，PC4控制
      • INTEB，B 口中断允许寄存器，PC2控制。PC2置为1，中断允许
        • 似乎是要看附录里的图，认准偶数下标的 PC 端
    • 方式 1：选通型输入输出方式
      • 选通型输入输出方式，端口和外设之间必须有联络线，CPU 与 8255A 可以用查询方式或中断方式交换信息
      • 联络信号：
        • IBFA\rm IBF_AIBFA​，Input Buffer Full，输入缓冲器满。为 1 时可继续 IN 读入外设数据
        • OBFA‾\rm \overline{OBF_A}OBFA​​，输入缓冲器满。为 1 时可用 OUT 输出数据给外设
      • A口、B口 适用
      • INTEA，A 口中断允许寄存器，PC6控制
      • INTEB，B 口中断允许寄存器，PC2控制
    • 方式 2：双向数据传输方式
      • 双向数据传输方式
      • A口 适用
      • INTE1，A 口输出中断允许寄存器，PC6控制
      • INTE2，B 口输入中断允许寄存器，PC4控制
  • 初始化编程，分为两步
    • 方式选择控制字，写到控制口
      • D7 = 1，表示是在设置方式
    • 允许中断（或禁止中断）的命令字，写到控制口。方式 0 不需要这步
      • D7~D4 = 0000

第十一章 可编程定时器/计数器

8254

  • 内部结构

    • 8254 包含的部分

      • 集成了 3 个独立的 16 位计数器，各有 6 种工作方式
        • 0# 计数器， 初值为 0，OUT0 是周期为 55ms 的方波信号。每隔 55ms 通过系统中断提出一次日时钟中断请求
        • 1# 计数器，动态存储器刷新请求。用户程序不可用
        • 2# 计数器，为音响提供 900Hz 方波
      • 数据总线缓冲器、读写控制器、控制字寄存器、3 个计数器
      • 引脚：
        • 有三组的 GATE CLK OUT，分别对应 0 ~ 2 号计数器
        • GATE 为 1 时，开始计数

    • 计数器的初值：fclk/foutf_{clk}/f_{out}fclk​/fout​。即，输出的一个计数周期总在一个输入周期以内完成

    • 数据总线缓冲器，8 位寄存器，对外有 8 条数据线 D7∼0\rm D_{7\sim0}D7∼0​

      • CPU 写入命令字，控制 8254 工作方式；
      • 向某一计数器写入计数初值；
      • 从某一计数器读取当前计数值

    • 读写控制模块，选择控制存储器和计数器，

      • 对外有 5 条引线：
        • 片选 CS‾\overline{\rm CS}CS。为 1 时，芯片工作；为 0 时，与 CPU 脱开
        • 由 A1A_1A1​ A0A_0A0​ 信号选择内部寄存器
        • RD‾\overline{\rm RD}RD 和 WR‾\overline{\rm WR}WR 决定寄存器是读还是写

    • 控制字寄存器，在初始化编程时写入控制字，决定计数器的工作方式

      • 只能写入，不能读出

    • 3 个计数器

      • 对外有 2 条线：
        • GATE，控制信号输入。为 1 时，CLK 下降沿使计数器减 1，为 0 则不处理。减到 0 时 OUT 输出信号
        • CLK，计数脉冲输入
        • OUT，计数器输出
  • 8254 有 6 种工作方式：
    • 每个计数器有 6 种工作方式，均在 CLK 下降沿计数一次
    • 方式 2：分频器（脉冲发生器）。产生一个连续的负脉冲信号
      • 高电平 : 低电平 = N - 1 : 1。其中，N 是初值
    • 方式 3：方波发生器
      • 减 2 计数，减到 0 或小于 0 时跳变并重装
      • 所以，N 为偶数时，高电平 : 低电平 = 1 : 1
  • 8254 有 2 种启动方式：
    • 硬件启动、软件启动
  • 8254 的初始化编程
    • 方式选择命令字，写入到控制口
    • 计数初值，写入到选择的计数器
