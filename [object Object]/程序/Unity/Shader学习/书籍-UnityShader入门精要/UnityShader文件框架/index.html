Unity Shader 的结构

shader文件内的代码框架示例如下。

Properties

材质面板上需要显示设置的内容都写在这里。

语法格式

[Attribute]_Name (“Display Name”, Type) = Default Value

  • Attribute
    属性标记。可选

    • [NoScaleOffset]，不允许在材质面板上设置纹理的偏移
    • [Header(Header Name)]，在材质面板上，分组
    • [HideInInspector]，在材质面板上隐藏此参数

  • _Name
    变量名。必须加下划线

  • Display Name
    显示在材质面板上的名称，起到说明作用

  • Type
    属性的类型。常用类型如下：

    属性类型 默认值的定义语法 例子
    Int number _Int(“Int”,Int)=2
    Float number _Float(“Float”,Float)=1.5
    Range(min,max) number _Range(“Range”,Range(0.0, 5.0))=3.0
    Color (number,number,number,number) _Color(“Color”,Color)=(1,1,1,1)
    Vector (number,number,number,number) _Vector(“Vector”,Vector)=(2,3,6,6)
    2D “defaulttexture”{} _2D(“2D”,2D)=“”{}
    Cube “defaulttexture”{} _Cube(“Cube”,Cube)=“white”{}
    3D “defaulttexture”{} _3D(“3D”,3D)=“black”{}
    • Range，隐性代表Float，在材质面板上生成一个滑条。用法示例：Range(0, 1)。
      • 可以配合属性标记[PowerSlider(n)]来控制滑条精度
      • 若存在属性标记[IntRange]，滑条只能固定在整数值上
      • 若存在属性标记[Toggle]，会在 0 和 1 中设置

  • Default Value
    默认值

SubShader

每个 Unity Shader 文件可以包含多个 SubShader 块，至少一个。
其定义通常如下，

[RenderSetup]（状态设置）

可以设置显卡状态，例如是否开启混合/深度测试、是否进行背面剔除。

名称 设置指令 含义
cull cull back/front/off 设置剔除模式：剔除背面（默认）、剔除正面、不剔除
ztest ztest less greater/lequal/gequal/equal/notequal/always 深度测试时使用的函数
zwrite zwrite on/off 是否开启深度写入
blend blend srcfactor/dstfactor 开启并设置混合模式

[Tags]（标签）

是一个字符串类型键值对，告诉 Unity 渲染引擎如何渲染对象。

标签 说明 例子
Queue 指定该物体属于哪一个渲染队列，控制渲染顺序 Tags{“Queue”=“Transparent”}
RenderType 对着色器进行分类，用于着色器替换（Shader Replacement） Tags{“RenderType”=“Opaque”}
DisableBatching 指明是否对该 SubShader 使用批处理，避免某些情况下的问题 Tags{“DisableBatching”=“True”}
ForceNoShadowCasting 使用该 SubShader 的物体是否会投射阴影 Tags{“ForceNoShadowCasting”=“True”}
IgnoreProjector 物体是否受到 Projector 影响。通常用于半透明物体 Tags{“IgnoreProjector”=“True”}
CanUseSpriteAtlas 若 SubShader 用于精灵 Sprites，将此标签设为 False Tags{“CanUseSpriteAtlas”=“False”}
PreviewType 如何预览该材质。默认为球形 Tags{“PreviewType”=“Plane”}

Pass

顶点着色器和片段着色器。

首先，框架如下：

  • [Name]，若是写成 Name "MyPass"，通过 UsePass "Example/SomeShader/MYPASS"，其他 SubShader 可以使用这段 Pass

  • [Tag]，类似 SubShade 的标签

    标签 说明 例子
    LightMode 该 Pass 在 Unity 的渲染流水线中的角色 Tags{“LightMode”=“ForwardBase”}
    RequireOptions 满足某些条件时才渲染这个 Pass Tags{“RequireOptions”=“SoftVegetation”}

两个特殊的 Pass 段：

  • UsePass，使用其它 Unity Shader 中的 Pass
  • GrabPass，抓取屏幕并将结果存储在一张纹理中，以用于后续的 Pass 处理

Cg/HLSL 中的数据类型

ShaderLab 中属性的类型和 Cg 中变量的类型之间的匹配关系如下：

ShaderLab 属性类型 Cg 变量类型
Color, Vector float4, half4, fixed4
Range, Float float, half, fixed
2D sampler2D
Cube samplerCube
3D sampler3D

可选的 Fallback 指令。若没有可以运行的 SubShader，就会使用指定的 shader。
