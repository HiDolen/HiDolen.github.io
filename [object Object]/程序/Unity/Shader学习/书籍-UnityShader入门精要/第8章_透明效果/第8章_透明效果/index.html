前言

在 Unity 中，通常使用两种方法实现透明效果：

  • 透明度测试（Alpha Test），无法得到真正的半透明效果
    • 对于要透明处理的片元，若其透明度不满足条件（通常是小于某个阈值）就会被直接舍弃处理，否则就会按照普通不透明物体的方式来处理
    • 意思是，要么看不见，要么就完全不透明
  • 透明度混合（Alpha Blending）

    • 与已经存储在颜色缓冲中的颜色值进行混合，得到真正的半透明效果

    • 透明度混合需要关闭深度写入（并没有关闭深度测试，这个片元仍然受深度值的影响）

    • 因为关闭了深度写入，透明物体的渲染次序开始产生巨大作用：

      • 透明物体 应在 不透明物体 之后渲染
      • 距离摄像头更近 的透明物体应在 距离摄像头更远 的透明物体之后渲染

    • 即使控制了渲染次序，有些情况下仍然会出错

    • 解决这种循环覆盖情况的方法，需要写两个 Pass。在后文详细说明

Unity Shader 的渲染顺序

渲染队列（render queue），可以使用 SubShader 的 Queue 标签来决定模型归于哪个渲染队列。

Unity 使用一系列整数索引表示各个渲染队列，索引号越小越早被渲染。

Properties Value 渲染队列描述
Background 1000 这个队列通常被最先渲染（比如天空盒）
Geometry 2000 默认渲染队列，不透明几何体使用该队列
AlphaTest 2450 需要开启透明度测试的物体
GeometryLast – 最后一个被认为是不透明的渲染队列
Transparent 3000 透明度混合物体使用该队列（例如玻璃和粒子效果），按照从后往前的顺序进行渲染
Overlay 4000 实现叠加效果，适合最后渲染的物体（如镜头光晕）

ShaderLab 的混合命令

为了进行混合，需要使用 Unity 提供的混合命令 Blend。

混合和两个操作数有关：源颜色（source color）目标颜色（destination color），分别用 S 和 D 表示。源颜色是指由片元着色器产生的颜色值，目标颜色是指从颜色缓冲中读取到的颜色值。全过程的颜色都有 RGBA 四个通道。

  • Blend Off，关闭混合
  • Blend SrcFactor DstFactor，开启混合，并设置 RGB 通道的混合因子
    • 源颜色（该片元产生的颜色）会乘以 SrcFactor，而目标颜色（已经存在于颜色缓存的颜色）会乘以 DstFactor，然后把两者相加后存入颜色缓冲中
    • 一般使用 Blend SrcAlpha OneMinusSrcAlpha。意义是 alpha 乘源颜色，(1-alpha) 乘目标颜色，再相加
值 功能
One 此输入的值是 one。该值用于使用源或目标的颜色的值。
Zero 此输入的值是 zero。该值用于删除源或目标值。
SrcColor GPU 将此输入的值乘以源颜色值。
SrcAlpha GPU 将此输入的值乘以源 Alpha 值。
DstColor GPU 将此输入的值乘以帧缓冲区的源颜色值。
DstAlpha GPU 将此输入的值乘以帧缓冲区的源 Alpha 值。
OneMinusSrcColor GPU 将此输入的值乘以（1 - 源颜色）。
OneMinusSrcAlpha GPU 将此输入的值乘以（1 - 源 Alpha）。
OneMinusDstColor GPU 将此输入的值乘以（1 - 目标颜色）。
OneMinusDstAlpha GPU 将此输入的值乘以（1 - 目标 Alpha）。
  • Blend SrcFactor DstFactor, SrcFactorA DstFactorA，设置透明通道 A 通道的混合因子
  • BlendOp BlendOperation，改变源颜色和目标颜色的运算（默认为相加）
值 功能
Add 将源和目标相加（默认）
Sub 从源减去目标
RevSub 从目标减去源
Min 使用源和目标中的较小者，逐分量比较，混合因子没有效果
Max 使用源和目标中的较大者，逐分量比较，混合因子没有效果
