Unity的内置内容

发表于2022-09-08|更新于2022-09-08|程序UnityShader学习书籍-UnityShader入门精要
|阅读量:

一些函数与变量

  • UnityObjectToClipPos()，将顶点坐标从 模型空间 转换到 裁剪空间

    与 mul(UNITY_MATRIX_MVP,*) 等价。
    使用例：

  • reflect(i, n) ，计算反射方向。i 是入射方向，n 是法线方向。可以是 float、float2、float3

  • saturate(x)，把 x 截取在 [0, 1] 范围内。如果 x 是个矢量，那么会对每个分量都操作一遍。

  • unity_WorldToObject()，等价于 _World2Object

  • mul(a, b)，矩阵相乘。数组（如 fixed4）会根据在参数的位置自动设置为行矩阵或列矩阵

  • TRANSFORM_TEX()，对纹理进行缩放和偏移处理。#define TRANSFORM_TEX(tex,name) (tex.xy * name##_ST.xy + name##_ST.zw)

  • floor()，向下取整

  • frac()，获得小数部分

Unity 的包含文件

包含文件（include file），类似 C 语言的头文件，后缀为 .cginc。可以通过形如 #include "UnityCG.cginc" 的方式将文件包含进来。

CGIncludes 中主要包含的文件及用处如下：

文件名 描述
UnityCG.cginc 包含了最常用的帮助函数、宏和结构体等
UnityShaderVariables.cginc 在编译 Unity Shader 时会自动包含进来。包含了诸如 UNITY_MATRIX_MVP 的内置全局变量
Lighting.cginc 包含了各种内置的光照模型。如果编写的是 Surface Shader，会自动包含进来
HLSLSupport.cginc 声明了很多用于跨平台编译的宏和定义。在编译 Unity Shader 时会被自动包含进来

UnityCG.cginc

文件可以在 Unity 目录下的 Editor->Data->CGIncludes 里面找到。

包含的宏：

  • TRANSFORM_TEX，快捷地对纹理进行偏移操作
    • #define TRANSFORM_TEX(tex, name) (tex,xy * name##_ST.xy + name##_ST.zw)
  • TANGENT_SPACE_ROTATION，构建从切线空间到世界空间的变换矩阵 
    • #define TANGENT_SPACE_ROTATION \
float3 binormal = cross(normalize(v.normal), normalize(v.tangent.xyz)) * v.tangent.w; \
float3x3 rotation = float3x3( v.tangent.xyz, binormal, v.normal )

其包含的一些结构体的名称和包含的变量：

名称 描述 包含的变量
appdata_base 可用于顶点着色器的输入 顶点位置、顶点法线、第一组纹理坐标
appdata_tan 可用于顶点着色器的输入 顶点位置、顶点切线、顶点法线、第一组纹理坐标
appdata_full 可用于顶点着色器的输入 顶点位置、顶点切线、顶点法线、四组（或更多）纹理坐标
appdata_img 可用于顶点着色器的输入 顶点位置、第一组纹理坐标
v2f_img 可用于顶点着色器的输出 裁剪空间中的位置、纹理坐标

包含的部分帮助函数：

函数名 描述
float3 WorldSpaceViewDir(float4 v) 输入一个模型空间中的顶点位置，返回世界空间中从该点到摄像机的观察方向
float3 ObjSpaceViewDir(float4 v) 输入一个模型空间中的顶点位置，返回模型空间中从该点到摄像机的观察方向
float3 WorldSpaceLightDir(float4 v) 仅可用于前向渲染中。输入一个模型空间中的顶点位置，返回世界空间中从该点到光源的光照方向。没有被归一化
float3 ObjSpaceLightDir(float4 v) 仅可用于前向渲染中。输入一个模型空间中的顶点位置，返回模型空间中从该点到光源的光照方向。没有被归一化
float3 UnityObjectToWorldNormal(float3 norm) 把法线方向从模型空间转换到世界空间中
float3 UnityObjectToWorldDir(float3 dir) 把方向矢量从模型空间转换到世界空间中
float3 UnityObjectToObjectDir(float3 dir) 把方向矢量从模型空间转换到模型空间中

语义

语义（semantics），让 Shader 知道从哪里读取数据、数据输出到哪里。

系统数值语义（system-value semantics），这类语义是以 SV 开头的。用这些语义描述的变量不可被随便赋值，因为流水线需要使用它们来完成特定目的。为了让编写的 Shader 能更好地跨平台，对于某些具有特殊含义的变量，最好优选选用以 SV 开头的语义进行修饰。

Unity 支持的语义

从应用阶段传递模型数据给顶点着色器（a2v）时的常用语义：

语义 描述
POSITION 模型空间中的顶点位置，通常是 float4 类型
NORMAL 顶点法线，通常是 float3 类型。a2v 中得到的顶点法线是位于模型空间下的，可以使用 mul(v.normal, (float3x3)unity_WorldToObject) 转换到世界空间
TANGENT 顶点切线，通常是 float4 类型
TEXCOORDn，如 TEXCOORD0 该顶点的纹理坐标，TEXCOORD0 表示第一组纹理坐标，依此类推。通常是 float2 或float4 类型
COLOR 顶点颜色，通常是 fixed4 或 float4 类型

从顶点着色器传递数据给片元着色器（v2f）时的常用语义：

语义 描述
SV_POSITION 裁剪空间中的顶点坐标，SV 指 Systems Value。结构体中必须包含一个用该语义修饰的变量
COLOR0 通常用于输出第一组顶点颜色，不是必须的
COLOR1 通常用于输出第二组顶点颜色，不是必须的
TEXCOORD0 ~ TEXCOORD7 通常用于输出纹理坐标，不是必须的

片元着色器输出时的常用语义：

语义 描述
SV_Target 输出值将会存储到渲染目标（render target）

Shader 的语义差异

为了让 Shader 能够在所有平台上正常工作，应尽可能使用以下语义来描述 Shader 的输入输出变量：

  • 使用 SV_POSITION 描述顶点着色器输出的顶点位置
  • 使用 SV_Target 来描述片元着色器的输出颜色
