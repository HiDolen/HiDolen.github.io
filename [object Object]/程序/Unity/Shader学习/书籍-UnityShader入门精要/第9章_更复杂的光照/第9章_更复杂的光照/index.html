Unity 的渲染路径

渲染路径（Rendering Path）决定了光照如何应用到 Unity Shader 中。例如，Pass 获得光照信息的变量。

设置整个项目渲染时的渲染路径：Edit → Project Settings → Player → Other Setting → Rendering Path。
设置单个摄像机的渲染路径：Inspector → Rendering Path。

  • 前向渲染路径（Forward Rendering Path），将使用 ForwardBase 和 ForwardAdd 通道
  • 延迟渲染路径（Deferred Rendering Path），将使用 Deferred 通道
  • 旧版延迟光照路径
  • 旧版顶点光照渲染路径，若使用向前渲染但着色器没有适合向前渲染的通道（即，ForwardBase 和 ForwardAdd 通道类型均不存在），则会按照在顶点光照通道中的方式来渲染该对象

LightMode 确定渲染路径

指定 Pass 使用的渲染路径，使用 LightMode 标签实现。例如：

LightMode 标签支持的渲染路径设置：

标签名 描述
Always 始终渲染，不应用任何光照（默认）
ForwardBase 在前向渲染中使用。该 Pass 会计算环境光、主方向光、顶点/SH 光源和光照贴图
ForwardAdd 在前向渲染中使用。该 Pass 会计算额外的逐像素光源，每个 Pass 对应一个光源
Deferred 在延迟渲染中使用。渲染 G 缓冲区（G-buffer）
ShadowCaster 将对象的深度信息渲染到阴影贴图（shadowmap）或深度纹理中
MotionVectors 用于计算每个对象的运动矢量
PrepassBase 用于旧版延迟光照。渲染法线和镜面反射指数
PrepassFinal 用于旧版延迟光照。通过组合纹理、光照和发光来渲染最终颜色
ShadowCaster 将对象深度渲染到阴影贴图或深度纹理中
Vertex VertexLMRGBM VertexLM 用于旧版顶点光照渲染（当对象不进行光照贴图时）
Meta 在常规渲染期间不使用此通道，仅用于光照贴图烘焙或实时全局光照

前向渲染路径

Unity 中有 3 种处理光照的方式：逐顶点处理、逐像素处理、球谐函数（Spherical Harmonics, SH）处理。

光源渲染模式（Render Mode），可以设置光源是否重要（Important）。重要的光源会被当成一个逐像素光源来处理，剩下的可以按 SH 方式处理。在光源的 Light 组件中对这个属性进行设置。

Edit → Project Settings → Quality 中的 Pixel Light Count 决定了逐像素光源数量。Additional Pass 处理除了平行光的其他逐像素光源。

最多有 4 个光源按逐顶点的方式处理。Unity 判断光源重要程度的规则：

  • 场景中最亮的平行光，总是逐像素处理
  • 渲染模式被设置为 Not Important 的光源，会按逐顶点或者 SH 处理
  • 渲染模式被设置为 Important 的光源，会按逐像素处理
  • 如果上述规则得到的逐像素光源数量小于 Pixel Light Count，会有更多光源以逐像素的方式渲染

计算光照的 Pass

前向渲染有两种 Pass：Base Pass 和 Additional Pass。

  • Base Pass
    • 使用标签 Tags{"LightMode"="ForwardBase"}
    • 使用编译指令 #pragma multi_compile_fwdbase
    • 可实现的光照效果：光照纹理，环境光，自发光，阴影（平行光的阴影）
    • 光照计算：一个逐像素的平行光，以及所有逐像素点和SH光源
  • Additional Pass
    • 使用标签 Tags{"LightMode"="ForwardAdd"}
    • 使用混合模式 Blend One One，否则会覆盖之前的渲染结果
    • 使用编译指令 #pragma multi_compile_fwdbase。若要为点光源和聚光灯开启阴影效果，将编译指令换为 #pragma multi_compile_fwdadd_fullshadows

注意事项：

  • 每次前向渲染，一个 Base Pass 仅执行一次，而 Additional Pass 会根据影响物体的其他逐像素光源的数目被多次调用，即每个逐像素光源会执行一次 Additional Pass
  • 环境光和自发光只需要计算一次，所以这些计算都在 Base Pass 里处理
  • 以上提到的在哪个 Pass 计算什么光照是通常做法。Unity 提供的内置变量如何处理完全取决于开发者

内置的光照变量和函数

unity官方文档

前向渲染（即 LightMode 为 ForwardBase 或 ForwardAdd）可使用的变量：

名称 类型 描述
_LightColor fixed4 该 Pass 处理的逐像素光源的颜色
_WorldSpaceLightPos0 float4 方向光：（世界空间方向，0）；其他光源：（世界空间位置，1）
unity_4LightPosX0 unity_4LightPosY0 unity_4LightPosZ0 float4 （仅限 ForwardBase 通道）前四个非重要点光源的世界空间位置
unity_4LightAtten0 float4 （仅限 ForwardBase 通道）前四个非重要点光源的衰减因子。
unity_LightColor half4[4] （仅限 ForwardBase 通道）前四个非重要点光源的颜色。

前向渲染可使用的函数：

函数名 描述
float3 WorldSpaceLightDir(float4 v) 仅可用于前向渲染中。输入一个模型空间中的顶点位置，返回世界空间中从该点到光源的光照方向，即以该点为起点。内部实现使用了 UnityWorldSpaceLightDir 函数。没有被归一化
float3 UnityWorldSpaceLightDir(float4 v) 仅可用于前向渲染中。输入一个世界空间中的顶点位置，返回世界空间中从该点到光源的光照方向。没有被归一化
float3 ObjSpaceLightDir(float4 v) 仅可用于前向渲染中。输入一个模型空间中的顶点位置，返回模型空间中从该点到光源的光照方向。没有被归一化
float3 Shade4PointLights(...) 仅可用于前向渲染中。计算四个点光源的光照，参数是已经打包进矢量的光照数据。前向渲染通常用这个函数计算逐顶点光照

延迟渲染路径

除了颜色缓冲和深度缓冲，延迟渲染会利用额外的 G 缓冲（G-buffer）（Geometry），存储了表面的法线、位置、用于光照计算的材质属性等。

延迟渲染主要包含了两个 Pass。第一个 Pass 不进行任何光照计算，仅计算哪些片元是可见的（运用深度缓冲技术）。若一个片元可见，就会把它的相关信息存储到 G 缓冲区中。之后，在第二个 Pass 中，利用 G 缓冲区中的各个片元信息，进行光照计算。

可见，延迟渲染的效率不依赖于场景的复杂度，而是挂钩于屏幕空间的大小。优化了性能。

延迟渲染具有的缺点：

  • 不支持真正的抗锯齿（anti-aliasing）功能
  • 不能处理半透明物体
  • 对显卡有一定要求。显卡必须支持 MRT（Multiple Render Targets）、Shader Mode 3.0 及以上、深度渲染纹理以及双面的模板缓冲

默认的 G 缓冲区包含了以下几个渲染纹理（Render Texture，RT）：

  • RT0，ARGB32 格式：漫射颜色 (RGB)，遮挡 (A)。
  • RT1，ARGB32 格式：镜面反射颜色 (RGB)，粗糙度 (A)。
  • RT2，ARGB2101010 格式：世界空间法线 (RGB)，未使用 (A)。
  • RT3，ARGB2101010（非 HDR）或 ARGBHalf (HDR) 格式：发射 + 光照 + 光照贴图 + 反射探针缓冲区。
  • 深度 + 模板缓冲区。

当在第二个 Pass 中计算光照时，默认情况下仅可以使用 Unity 内置的 Standard 光照模型。若要使用其他光照模型，需要替换原有的 Internal-DeferredShading.shader 文件。

内置的变量

名称 类型 描述
_LightColor float4 光源颜色
unity_WorldToLight float4x4 从世界空间到光源空间的变换矩阵。可以用于采样 cookie 和光强衰减纹理

Unity 的光源类型

Unity 一共支持 4 种光源类型：平行光、点光源、聚光灯、面光源（area light）。

最常用的 5 种光源属性：位置、方向、颜色、强度、衰减。

Unity 的光照衰减

获得衰减值的计算量较大，Unity 选择使用一张纹理作为查找表。用来查找的纹理变量为 _LightTexture0，用光源空间下的顶点坐标进行采样。使用 unity_WorldToLight 进行顶点转换，从世界空间转换到光源空间。

Unity 的阴影

实现阴影，Unity 使用名为 Shadow Map 的技术，大致流程是将摄像机放在光源位置，摄像机看不见的地方就是阴影。Unity 会计算光源的阴影映射纹理（shadowmap），记录了距离光源位置最近的表面位置。

为了更新光源的阴影映射纹理，Unity 会调用 Shader 中 LightMode 为 ShadowCaster 的 Pass。这个 Pass 的渲染目标不是帧缓存，而是阴影映射纹理（或深度纹理）。

综上，

  • 一个物体想要接受来自其他物体的阴影：
    • 在 Shader 中对阴影映射纹理进行采样，把采样结果和最后的光照结果相乘，产生阴影效果
  • 一个物体想要向其他物体投射阴影：
    • 把该物体加入到光源的阴影映射纹理的计算中。在 Shader 中执行 LightMode 为 ShadowCaster 的 Pass

额外注意事项：

  • 在光源的 Light 组件中，Shadow Type 一栏开启阴影，使得光源可以收集阴影信息
  • 在物体的 Mesh Renderer 组件中，设置 Cast Shadows 和 Receive Shadows 属性
    • 开启 Cast Shadows 属性，Unity 会把该物体加入到光源的阴影应黑色纹理的计算中（即执行 LightMode 为 ShadowCaster 的 Pass）
  • 若 Shader 中有 Fallback "Specular" 这样的回调，则可以调用回调里的 Pass，而不需要这个 Shader 里存在指定的 Pass
  • 默认情况下光源的阴影映射纹理会剔除物体的背面。若需要两面渲染，需要在物体的 Mesh Renderer 设置 Cast Shadows 属性为 Twe Sided

Shader 中阴影的宏

首先，为了能使用计算阴影的宏，需要 #include "AutoLight.cginc"。

  • 在 v2f 中使用宏 SHADOW_COORDS(2)，声明一个用于对阴影纹理采样的坐标
    • 参数 2 是插值寄存器的索引值（TEXCOORD2）
  • 在 vert 中使用宏 TRANSFER_SHADOW(o)，计算阴影纹理坐标，，即把顶点坐标从模型空间转换到光源空间
  • 在 frag 中用 fixed shadow = SHADOW_ATTENUATION(i); 进行纹理采样，获得阴影值

为了能正常使用宏，a2v 结构体的顶点坐标变量必须命名为 vertex，顶点着色器的输入结构体 a2v 必须命名为 v，v2f 中的顶点位置变量必须命名为 pos。

对于衰减值 atten，可以使用 Unity 内置宏（在 AutoLight.cginc 中被声明）：UNITY_LIGHT_ATTENUATION(atten, i, i.worldPos);

  • 第一个参数是衰减值的名称（这个宏会帮忙声明变量）
  • 这个衰减值会调用 SHADOW_ATTENUATION(i)，使得 atten 受阴影值影响
  • 第三个参数，用来计算光照衰减

使用这些内置宏的 Shader，详见文件 Chapter9-AttenuationAndShadowUseBuildInFunctions.shader

透明物体的阴影投射

对于透明度测试，只需要在 Fallback 部分写成 Fallback "Transparent/Cutout/VertexLit" 即可。

对于透明度混合，即使改成了 FallBack "Transparent/VertexLit"，也没有阴影，也不产生阴影。Unity 认为这样太消耗性能了，所以所有内置的半透明 Shader 都不会产生阴影效果。
