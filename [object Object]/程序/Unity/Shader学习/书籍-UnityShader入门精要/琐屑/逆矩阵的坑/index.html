_World2Object 与 normal

首先说，_World2Object 已经被新版本 Unity 替代了，成了 unity_WorldToObject，请注意。

_World2Object 矩阵是 _Object2World 矩阵的逆矩阵。以下代码是等价的：

  • o.worldNormal = mul(v.normal, (float3x3)unity_WorldToObject);
  • o.worldNormal = mul(transpose((float3x3)unity_WorldToObject), v.normal);

第一行代码的置换矩阵在右侧，一般转换矩阵在左侧。

为了避免等比缩放矢量，保证矢量垂直，需要这样做。

设 unity_ObjectToWorld 为 MMM，n⃗\vec{n}n 转世界空间矩阵为 GGG，我们最终的目的是希望转换后的 t′⃗\vec{t^{'}}t′ 与 n′⃗\vec{n^{'}}n′依旧保持垂直关系，所以

Mt⃗⋅Gn⃗=0M\vec{t}\cdot G\vec{n}=0Mt⋅Gn=0,

用矩阵的写法表示

t′⃗⋅n′⃗=(MT)⋅(GN)=0\vec{t^{'}}\cdot \vec{n^{'}}=(MT)\cdot (GN)=0t′⋅n′=(MT)⋅(GN)=0 (向量转成列矩阵形式,用其大写字母表示)

因为t′⃗⋅n′⃗\vec{t^{'}}\cdot \vec{n^{'}}t′⋅n′用矩阵表示为TTNT^{T}NTTN，所以

TTN=(MT)⋅(GN)=(MT)T(GN)=TTMTGNT^{T}N=(MT)\cdot (GN)=(MT)^{T}(GN)=T^{T}M^{T}GNTTN=(MT)⋅(GN)=(MT)T(GN)=TTMTGN

(注意运算符号从点乘变成了矩阵乘法,两个向量点乘相当于第一个矩阵转置乘以第二个矩阵)

所以 MTG=IM^{T}G=IMTG=I

所以G=(MT)−1=(M−1)TG=(M^{T})^{-1}=(M^{-1})^{T}G=(MT)−1=(M−1)T，可见 MMM 为正交矩阵

这里简单证明一下该性质
因为  (MM−1)T=(M−1)TMT=I(MM^{-1})^{T}=(M^{-1})^{T}M^{T}=I(MM−1)T=(M−1)TMT=I
所以 (M−1)T=(MT)−1(M^{-1})^{T}=(M^{T})^{-1}(M−1)T=(MT)−1

因为 M = unity_ObjectToWorld，所以M-1 = unity_WorldToObject

所以 G = unity_WorldToObject的转置，代码写作 transpose((float3x3)unity_WorldToObject)
